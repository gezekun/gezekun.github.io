python之禅

The Zen of Python, by Tim Peters

Beautiful is better than ugly.
Explicit is better than implicit.
Simple is better than complex.
Complex is better than complicated.
Flat is better than nested.
Sparse is better than dense.
Readability counts.
Special cases aren’t special enough to break the rules.
Although practicality beats purity.
Errors should never pass silently.
Unless explicitly silenced.
In the face of ambiguity, refuse the temptation to guess.
There should be one– and preferably only one –obvious way to do it.
Although that way may not be obvious at first unless you’re Dutch.
Now is better than never.
Although never is often better than right now.
If the implementation is hard to explain, it’s a bad idea.
If the implementation is easy to explain, it may be a good idea.
Namespaces are one honking great idea – let’s do more of those!

Python之禅 by Tim Peters

优美胜于丑陋（Python 以编写优美的代码为目标）
明了胜于晦涩（优美的代码应当是明了的，命名规范，风格相似）
简洁胜于复杂（优美的代码应当是简洁的，不要有复杂的内部实现）
复杂胜于凌乱（如果复杂不可避免，那代码间也不能有难懂的关系，要保持接口简洁）
扁平胜于嵌套（优美的代码应当是扁平的，不能有太多的嵌套）
间隔胜于紧凑（优美的代码有适当的间隔，不要奢望一行代码解决问题）
可读性很重要（优美的代码是可读的）
即便假借特例的实用性之名，也不可违背这些规则（这些规则至高无上）
不要包容所有错误，除非你确定需要这样做（精准地捕获异常，不写 except:pass 风格的代码）
当存在多种可能，不要尝试去猜测
而是尽量找一种，最好是唯一一种明显的解决方案（如果不确定，就用穷举法）
虽然这并不容易，因为你不是 Python 之父（这里的 Dutch 是指 Guido ）
做也许好过不做，但不假思索就动手还不如不做（动手之前要细思量）
如果你无法向人描述你的方案，那肯定不是一个好方案；反之亦然（方案测评标准）
命名空间是一种绝妙的理念，我们应当多加利用（倡导与号召）

第二章 变量和简单数据类型

问题：如何将列表中的全部内容首字母大写？

bicycles = [‘trek’, ‘cannondale’, ‘redline’, ‘specialized’]
bicycles = ‘ ‘.join(bicycles).title().split()
print(bicycles)

jion()方法用于将序列中的元素以指定的字符连接生成一个新的字符串。

str.join(sequence)
sequence – 要连接的元素序列。

str = “-“;
seq = (“a”, “b”, “c”); # 字符串序列
print str.join( seq );

split()通过指定分隔符对字符串进行切片，如果参数 num 有指定值，则分隔 num+1 个子字符串

str.split(str=””, num=string.count(str))
str – 分隔符，默认为所有的空字符，包括空格、换行(\n)、制表符(\t)等。
num – 分割次数。默认为 -1, 即分隔所有。

str = “Line1-abcdef \nLine2-abc \nLine4-abcd”;
print str.split( ); # 以空格为分隔符，包含 \n
print str.split(‘ ‘, 1 ); # 以空格为分隔符，分隔成两个

第三章 列表

.append()是在列表末尾插入元素
.insert(0,’ ‘)，即可以在列表指定位置插入元素
del的用法不同，del list[0]
pop为弹出栈顶元素，list.pop()，也可以list.pop(1)，可弹出列表相应位置的元素
remove可以根据值删除列表元素，list.remove(‘**’)
sort()对列表进行永久排序，sorted()对列表进行临时排序，不会影响原列表的排序，也可(reverse=True)来进行反向排序
len()确定列表长度

第四章 列表操作

for循环可以遍历任何序列的项目，如一个列表或一个字符串
for magician in magicians:
即将magicians列表中的表一个元素依次传送到magician中
使用for要注意缩进
for magician in magicians：
print(magician.title() + “, that was a great trick !”)
则会依次打印出相应的语句，如果没有缩进则只会打印一次结束

range()可以生成一系列数字
range(1:5)即生成1，2，3，4四个数字，python中range()为左闭右开区间，即[)需要注意
list(range(1,5)),可生成[1,2,3,4]列表

列表解析可以让数行代码以简短的几行代码来表示，更简便也更加优雅
squares = []
for value in range(1,11):
square = value“”2 # 平方
squares.append(square)
print(square)
我们可以以下面的代码来表示
squares = [value“”2 for value on range(1,11)]
print(squares)

python中通常用切片，Python对指针做了良好封装，一切都是“对象”，一切对象都有一个“变量”指向它。这个“变量”就是“指针”。和Java一样，它不可以随机移动，不可以运算。包括函数也是一个“对象”，用一个变量指向它，就是指向函数的入口地址。
用print(list[0:3])来打印出列表前三个元素

加入for循环就可以遍历切片，遍历切片可以显示部分想要显示的信息
for player in players[:3]
print(player)
可以依次打印出列表的前三个元素
a_list = b_list[:]可以遍历b列表依次把元素复制到a列表中

列表是可以修改的，不可改变的列表成为元组
元组看起来像列表，但使用圆括号而不是放括号来标识，想要改变元组中的元素只能重新定义整个元组

第五章 if语句

布尔表达式
它是条件测试的别名，与条件表达式相同，布尔表达式的结果要么为True，要么为False。布尔表达式通常用于记录条件，如游戏是否在运行，或用户是否可以编辑网站的特定内容，如：
game_active = True
game_edit = False
在跟踪程序状态或程序中重要的条件方面，布尔值提供了一种高效的方式。

检测列表为空：
定义一个text_list = []
用
if text_list:
 print(“ “)

print(“ “)
即可进行列表是不是空的判断，当列表不为空时执行if中的输出，否则直接跳出判断进行输出。

第六章 字典

字典由键值对来构成，即 dictionaries = {‘key’:’value’}
遍历字典需要遍历所有的键值对，需要声明两个变量来分别存储键值对
for a_key, b_value in dictionaries.items():
 print(“\nKey: “ + a_key)
 print(“value: “ + b_value)
若遍历字典中的所有键
for a_key in dictionaries.keys():
 print(“\nKey: “ + a_key)
若遍历字典中的所有值
for b_value in dictionaries.values():
 print(“\nvalue: “ + b_value)

字典中也可以嵌套列表和字典

第七章 用户输入和while循环

输入：input(“想要输入的值: “)
int() 函数用于将一个字符串或数字转换为整型
str() 函数将对象转化为适于人阅读的形式

while循环会不断运行，直到指定的条件不满足时
for循环用于针对集合中的每个元素都有一个代码块
break可以强制跳出整个循环
continue 语句用来告诉Python跳过当前循环的剩余语句，然后继续进行下一轮循环
continue 语句跳出本次循环，而break跳出整个循环
